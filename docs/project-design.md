# AI 萌新小窝 - 项目整体设计文档

## 项目定位

**AI 萌新小窝** 是一个面向完全不懂编程和AI的纯新人的科普与创作平台。

**一句话定位**：用AI教新人读书和写作，让零基础的人也能享受文学创作的乐趣。

**内容优先级（最重要的四件事）**：

🥇 **第一优先级：AI拆书读书**（本项目最重要的目的）
- 用AI逐段拆解经典名著和热门小说
- 解析名著名片的手法结构、叙事技巧、人物塑造
- 重点：不是总结，是品味——逐字逐句分析写作技法

🥈 **第二优先级：AI写作创作**
- 用AI教新人写小说、文案、搞笑段子等
- 重点教AI写小说，这是本项目的核心差异化方向
- 让新人从"不会写"到"会创作"

🥉 **第三优先级：AI知识资讯**
- AI领域最新动态、科技新闻、行业趋势
- 适合新人的AI工具推荐和使用指南
- 创业思路、变现机会

🏅 **第四优先级：AI热门项目**
- GitHub热门AI开源项目解读
- 优质AI学习资源和工具库

## 目标用户

### 核心用户画像
- **纯新人**：完全不懂编程和AI，但对AI充满好奇的普通人
- **写作爱好者**：想用AI辅助创作小说、文案、段子等文学内容的创作者
- **职场新人**：希望通过AI提升工作效率，但不知从何入手的职场人士
- **学生群体**：对AI感兴趣，想了解最新科技动态的学生

### 用户痛点
1. AI概念太多太杂，不知道从哪里开始学
2. 网上AI教程偏技术向，新人看不懂
3. 知道AI能写东西，但不知道怎么用、用什么工具
4. 想学AI写小说，但找不到系统的方法和案例
5. 学了之后没有交流社区，孤军奋战容易放弃
6. 想提升写作水平，但不知道如何从经典作品中学习技法

## 重点方向：AI拆书读书

这是本项目的**第一优先级**，也是最重要的核心功能。

### 核心理念：拆书不是总结，是品味

不是用AI把一本书浓缩成几句话，而是让AI像一个文学导师，带着你逐字逐段地读，每一句话、每一个用词都要细细品味。把一本厚书拆成一个个独立的小段，每段都可以单独阅读、单独理解、单独欣赏。

### 拆书的核心目标：解析名著名片的手法结构

重点拆解以下内容：
- **叙事结构**：作品是怎么讲故事的，开头、中间、结尾如何安排
- **人物塑造**：作者是怎么让人物立体鲜活的，对话、动作、心理怎么写
- **场景描写**：怎么用文字构建画面感，细节怎么选择
- **语言风格**：作者的用词、句式有什么特点
- **伏笔悬念**：怎么埋下伏笔，怎么制造悬念

### 为什么拆书是第一优先级？
1. **读写闭环**：写好小说的前提是读好小说，拆书是写作的输入，写作是拆书的输出
2. **学习技法**：通过拆解经典段落，新人能直观地看到"好文字长什么样"
3. **降低门槛**：很多新人不敢写是因为不知道好作品好在哪里，拆书帮他们建立文学审美
4. **内容互补**：拆书内容天然适合持续更新，一部经典可以拆出几十期

### 拆书内容规划
| 拆书主题 | 内容要点 | 优先级 |
|----------|----------|--------|
| 《红楼梦》经典段落 | 黛玉葬花、宝钗扑蝶、刘姥姥进大观园等 | ⭐⭐⭐ |
| 《活着》经典段落 | 福贵买牛、家珍之死等 | ⭐⭐⭐ |
| 《百年孤独》经典段落 | 开头名句、冰块场景等 | ⭐⭐⭐ |
| 《围城》经典段落 | 方鸿渐归国、婚姻困境等 | ⭐⭐ |
| 余华笔法赏析 | 跨作品分析余华的极简叙事风格 | ⭐⭐ |
| 如何用AI拆书 | 拆书方法论和Prompt模板 | ⭐⭐⭐ |

### 拆书的独特价值
- **逐字逐句**：不是概括大意，而是每个词都要品味——为什么用这个词？换一个会怎样？
- **独立小段**：每期只拆一小段，即使没读过全书也能独立欣赏
- **技法提炼**：从经典段落中提炼出可操作的写作技法
- **AI辅助**：提供拆书Prompt模板，教新人学会自己用AI拆书

### 拆书与写作的联动
| 拆书内容 | 对应写作教程 | 联动方式 |
|----------|-------------|----------|
| 经典开头拆解 | AI小说开头写作 | 学完拆书后练习写开头 |
| 对话描写拆解 | AI对话写作技巧 | 学完拆书后练习写对话 |
| 场景描写拆解 | AI场景写作技巧 | 学完拆书后练习写场景 |
| 人物塑造拆解 | AI角色塑造教程 | 学完拆书后练习写人物 |

### 为什么做AI写作创作？
1. **用户需求强**：大量新人想学用AI写小说，这是AI创作中最有吸引力的方向
2. **内容壁垒高**：目前网上缺少系统化的AI小说写作教程，大部分是零散的技巧分享
3. **社区粘性强**：小说创作天然适合社区互动——分享作品、互相评论、连载追更
4. **商业价值大**：AI写作是AI变现的重要方向，用户学习动力足

## 技术架构

### 前端技术（最新流行技术栈，移动优先）
- **框架**：React 18 + Vite（快速开发和构建）
- **样式**：Tailwind CSS v4 + Shadcn UI（现代化组件库，移动优先设计）
- **状态管理**：React Query + Zustand（轻量级状态管理，适合移动设备性能）
- **路由**：React Router v6（支持移动端导航模式）
- **Markdown 渲染**：React Markdown + rehype-highlight（代码高亮，移动端优化）
- **响应式设计**：移动优先设计（Mobile-First），优先确保手机小屏幕显示效果
- **构建工具**：Vite（快速热更新，支持移动端开发体验）
- **移动端优化**：
  - 触摸友好的交互设计（大按钮、合理的点击区域）
  - 延迟加载（Lazy Loading）图片和内容
  - 离线支持（PWA 特性）
  - 字体大小自适应（rem 单位）
  - 触摸手势支持（滑动、捏合缩放等）

### 后端服务
- **框架**：Next.js 14（App Router，全栈解决方案）
- **API**：RESTful API + Server Actions（Next.js 原生功能）
- **认证**：NextAuth.js（支持邮箱、Google、GitHub 登录）
- **文件上传**：Vercel Blob（文件存储）
- **AI 集成**：OpenAI API + 其他大模型 API 集成

### 数据库
- **主数据库**：PostgreSQL（关系型数据库，支持复杂查询）
- **缓存**：Redis（提升性能）
- **存储**：Vercel Postgres（托管数据库服务）
- **数据结构**：
  - users（用户表）
  - posts（文章/拆书内容表）
  - comments（评论表）
  - likes（点赞表）
  - followers（关注表）
  - tags（标签表）

### 部署方案
- **托管平台**：Vercel（Serverless 部署，自动扩缩容）
- **域名**：使用 Vercel 提供的免费域名或自定义域名
- **CI/CD**：GitHub Actions + Vercel 自动部署
- **环境变量**：Vercel 环境变量管理
- **监控**：Vercel Analytics + Sentry（错误监控）

### 内容生成
- **Skill 系统**：`.trae/skills/` 目录下定义内容生成规则
- **每次生成6篇文章**，保持格式统一
- **自动化**：定时任务生成内容（使用 Vercel Cron Jobs）

### 创作者交流区（留言板升级版）
- 留言类型选择：问题咨询、作品分享、经验交流、建议反馈
- 不同类型用颜色标签区分
- 作品分享类型支持 Markdown 预览

### AI创作标签页
- 分类筛选按钮：全部、小说、文案、段子
- 文章卡片显示类型标签
- 底部引导创作者通过交流区分享作品
