# 精选项目与资源分享 —— 内容生成技能

## 模块定位

**精选项目** 是 AI 萌新小窝的六大核心模块之一，位于首页第一栏。该模块聚焦于向 AI 萌新推荐 GitHub 上的优质开源项目，帮助零编程基础的用户了解和挑选适合自己使用的 AI 工具与资源。

### 模块目标
- 让不懂编程的新人也能发现和理解 GitHub 上有价值的开源项目
- 用通俗易懂的语言介绍每个项目的功能、用途和使用场景
- 帮助用户建立对 AI 开源生态的基本认知
- 每个项目都附带个人评价，帮助新人判断"这个项目是否适合我"

## 内容方向与分类

### 一级分类：项目类型

| 分类 | 说明 | 推荐项目示例 | 优先级 |
|------|------|-------------|--------|
| **AI 提示词合集** | 收集实用 Prompt 的项目，帮助用户更好地与 AI 交互 | awesome-chatgpt-prompts | ⭐⭐⭐ |
| **编程语言资源** | 某种编程语言的框架、库、工具精选列表 | awesome-python | ⭐⭐⭐ |
| **AI 资源合集** | AI 领域的工具、框架、项目的精选列表 | awesome-ai | ⭐⭐⭐ |
| **AI 图像生成** | 基于 AI 的图像生成工具和项目 | stable-diffusion-webui | ⭐⭐ |
| **AI 应用框架** | 用于构建 AI 应用的框架和工具库 | LangChain | ⭐⭐ |
| **大模型部署** | 本地运行大语言模型的项目 | llama.cpp, Ollama | ⭐⭐ |

### 二级标签：项目特征
- **热门程度**：根据 Star 数分为超热门（100k+）、热门（10k-100k）、潜力（1k-10k）
- **上手难度**：零门槛（无需编程）、低门槛（需要简单配置）、中等（需要编程基础）
- **更新活跃度**：活跃维护、偶尔更新、归档项目

## 选题标准

每期 6 个项目的选取原则：

1. **新人友好优先**：优先选择零门槛或低门槛的项目，GitHub 仓库应提供清晰的使用说明
2. **Star 数要求**：原则上 Star 数不低于 1000，主流项目不低于 10000
3. **时效性**：选择最近 1 个月内仍在更新或 Star 数持续增长的项目
4. **多样性**：确保涵盖 AI 提示词、编程资源、AI 工具等多个分类，避免单一类型
5. **中文支持**：优先选择支持中文或社区有中文文档的项目
6. **表述通俗**：每个项目的介绍要用新人能理解的通俗语言，避免技术黑话

### 每周必选主题
- 至少 1 个 Awesome 系列项目（提示词合集或资源列表）
- 至少 1 个 AI 图像/视频生成项目
- 至少 1 个 AI 应用/开发框架项目

## 写作规范

### 语言风格
- **目标用户完全不懂编程**：所有描述用通俗易懂的日常语言，避免技术术语
- 如果必须使用术语，需在术语后加括号解释
- 多用类比和比喻帮助理解（例如："LangChain 就像一个乐高积木，你可以把不同的 AI 功能拼接起来"）

### 内容要求
- 每篇文章字数：正文部分（介绍 + 评价）不少于 300 字
- 一句话总结：必须是一句完整、通顺的中文句子，不含 Markdown 标记
- 详细介绍：不能只是翻译 GitHub README，要结合自己的理解进行解读
- 个人评价：不能是空洞的"非常好用"，要指出具体的优点、缺点和适用场景

### 禁止事项
- 禁止直接复制粘贴 GitHub README 内容
- 禁止在介绍中出现"详见 GitHub""请参考官方文档"等推脱性表述
- 禁止使用"后端""前端""API""SDK"等术语而不加解释

## 文件管理

### 文件命名规则
```
data/精选项目/{序号}-{项目简称}.md
```
示例：
- `001-awesome-chatgpt-prompts.md`
- `004-stable-diffusion-webui.md`

序号从 001 开始递增，每生成一轮自动编号。

### 目录结构
```
.trae/
  skills/
    featured-projects-skill/
      SKILL.md                    # 本文件：技能配置与规范
data/
  精选项目/
    001-awesome-chatgpt-prompts.md
    002-awesome-python.md
    003-awesome-ai.md
    ...
```

### 首页渲染
首页通过 `index.html` 中 `#section-articles` 标签页展示。每个项目卡片展示：项目名称、一句话总结、Star 数、查看入口。

## 使用方式

当用户请求"整理精选项目"或"推荐 GitHub 项目"时：
1. 从 GitHub Trending 或 Awesome 仓库中筛选 6 个符合条件的项目
2. 对每个项目进行调研（阅读 README、了解功能、评估适合度）
3. 按模板逐一生成项目解读文档
4. 保存到 `data/精选项目/` 目录
5. 告知用户文档已生成完成

---

## 注意事项

- Star 数会随时间变化，文档中标注的 Star 数以生成当日为准
- 如果项目已归档或停止维护，应在"缺点"中明确标注
- 确保所有链接可访问
- 项目名称使用 GitHub 官方名称，不做自行翻译
- 用户评价要体现"萌新视角"——以新手的角度评估易用性
