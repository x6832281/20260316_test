# GitHub 热门项目抓取工具

这个技能用于抓取GitHub上最近1个月最火的项目，生成6篇中文总结文章并保存到项目的精选项目文件夹。

## 功能说明

- **抓取来源**：GitHub Trending 页面
- **抓取内容**：最近1个月最火的项目
- **输出格式**：Markdown文档，保存到 `data/精选项目/` 目录
- **生成数量**：每次生成6篇中文总结文章

## 生成格式

每个项目的Markdown格式：

```markdown
# [项目名称]

**⭐ 星标**：XXX | **📅 更新日期**：YYYY-MM-DD

---

## 📝 一句话总结

[项目的一句话总结]

## 📋 详细介绍

[项目的详细介绍，包括功能、特点、使用方法等]

## 🎯 个人评价

[个人评价，包括项目的优缺点、使用体验、适合人群等]

## 🔧 技术栈

- [技术1]
- [技术2]
- [技术3]

## 🔗 相关链接

- [GitHub 仓库](链接)
- [项目官网](链接，如果有)
- [文档地址](链接，如果有)

---

**© 2026 AI Tools Magazine | AI 萌新小窝 出品**

[返回首页](/)
```

## 执行逻辑

1. **数据抓取**：从GitHub Trending页面获取最近热门项目
2. **数据处理**：提取项目名称、描述、Star数、Fork数、主要语言、更新日期等信息
3. **排序筛选**：根据Star数和更新时间筛选出最近1个月最火的项目
4. **生成文档**：为每个项目生成中文总结文章
5. **保存文件**：保存到 `data/精选项目/` 目录
6. **渲染到首页**：通过前端JavaScript自动读取 `data/精选项目/` 目录下的文件并渲染到首页

## 错误处理

- **网络请求失败**：自动使用预设的模拟数据
- **API限流**：添加请求间隔，避免被GitHub限流
- **文件保存失败**：输出错误信息并尝试重试

## 使用方式

当用户请求执行GitHub热门项目抓取时：
1. 调用此skill
2. 执行数据抓取和处理
3. 生成并保存项目总结文章
4. 告知用户文档已生成完成

```
      SKILL.md          # 技能配置文件

```

## 注意事项

- 抓取时会自动跳过无法访问的项目
- 每个请求间隔2秒，避免被GitHub限流
- 生成的文档会自动保存到项目根目录的 `data/精选项目/` 下
- 只抓取最近1个月内更新的项目
- 过滤掉Star数为0的项目
- 确保至少有2个awesome项目
