## 🚀 快速上手（5 分钟搞定）

### 方案一：使用云服务（推荐新手）

1. 访问 [Dify 官网](https://dify.ai/)
2. 注册账号
3. 创建应用
4. 开始使用

### 方案二：Docker 本地部署

```bash
# 克隆项目
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 启动服务
docker-compose up -d
```

访问 `http://localhost:3000` 即可使用。

### 方案三：手动部署

```bash
# 后端
cd api
pip install -r requirements.txt
flask run

# 前端
cd web
npm install
npm run dev
```

## 📚 核心功能详解

### 1. 知识库管理（最实用）

**使用场景**：建立专属写作素材库，让 AI 基于你的素材写作。

**操作步骤**：
1. 点击"知识库"→"创建知识库"
2. 上传文档（支持 PDF、Word、TXT、Markdown）
3. 等待系统自动向量化
4. 在应用中关联知识库

**实用技巧**：
- 把你的写作素材、参考资料、优秀案例都上传
- 定期更新知识库，保持素材新鲜
- 为不同主题创建不同知识库

### 2. 可视化工作流

**使用场景**：构建自动化写作流水线。

**操作步骤**：
1. 点击"创建应用"→"工作流"
2. 拖拽节点构建流程
3. 连接节点，设置输入输出
4. 测试并发布

**示例工作流**：
```
用户输入主题 → 搜索资料 → 分析素材 → 生成大纲 → 逐章写作 → 润色修改 → 输出成品
```

**实用技巧**：
- 先用简单流程测试，确认每步都正确
- 逐步增加复杂度
- 使用"条件分支"处理不同情况

### 3. 提示词工程

**使用场景**：管理和优化 AI 提示词。

**功能**：
- 可视化编辑提示词
- 支持变量和条件逻辑
- 版本管理，方便回滚
- A/B 测试，对比效果

**实用技巧**：
- 为不同写作场景创建不同提示词模板
- 使用变量让提示词更灵活
- 定期优化提示词，提升输出质量

### 4. AI Agent

**使用场景**：让 AI 自主完成复杂任务。

**功能**：
- 自动搜索资料
- 自动阅读文档
- 自动写作内容
- 自动润色修改

**操作步骤**：
1. 创建 Agent 应用
2. 配置工具（搜索、阅读、写作等）
3. 设置任务目标
4. 运行 Agent

## 💡 实战案例：自动写公众号文章

### 步骤一：创建知识库

1. 上传你的历史文章、参考资料
2. 等待系统向量化完成

### 步骤二：创建工作流

1. **输入节点**：用户输入文章主题
2. **搜索节点**：自动搜索相关资料
3. **分析节点**：提取核心观点

1. 输入测试主题
2. 查看每步输出
3. 调整提示词和参数
4. 反复测试直到满意

### 步骤四：批量生产

1. 准备主题列表
2. 批量运行工作流
3. 检查输出质量
4. 发布到公众号

## ⚠️ 使用注意事项

### 避坑指南

1. **学习曲线**
   - 可视化界面降低了门槛，但仍需理解概念
   - 建议先从简单应用开始

2. **成本控制**
   - 多步骤调用 API，费用会累积
   - 建议先用 DeepSeek 测试

3. **数据安全**
   - 云服务数据会经过第三方服务器
   - 敏感数据建议本地部署

4. **质量把控**
   - AI 生成的内容需要人工审核
   - 建立内容质量检查流程

## 🔧 常见问题解答

**Q：需要编程基础吗？**
A：不需要。Dify 是可视化界面，拖拽就能构建应用。但如果想深度定制，了解一些编程会更有帮助。

**Q：和 LangChain 有什么区别？**
A：LangChain 是开发者框架，需要写代码。Dify 是可视化平台，不懂代码也能用。功能上 Dify 更适合内容创作场景。

**Q：支持哪些大模型？**
A：支持 100+ 模型，包括 OpenAI GPT-4、Claude、DeepSeek、Qwen、GLM 等主流模型。

**Q：免费吗？**
A：云服务有免费额度，超出需要付费。本地部署完全免费，但需要自己准备服务器和 API Key。
