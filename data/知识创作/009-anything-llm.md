## 📌 一句话总结

**把任何文档变成 AI 知识库——全能型桌面 AI 应用，写作、研究、整理一站式搞定。**

## 🎯 这个工具能帮你解决什么问题？

如果你需要基于大量参考资料写作：

- 写作素材散落在各个文档里，每次都要手动复制粘贴
- 想让 AI 基于你的素材写作，但没有好的知识库方案
- 需要在不同项目之间切换，管理混乱
- 想在本地运行 AI，保护数据隐私

**AnythingLLM 就是为解决这些问题而生的。** 它是一个全能型桌面 AI 应用，把文档管理、知识库构建和 AI 对话深度整合。

## 🚀 快速上手（5 分钟搞定）

### 第一步：下载安装

1. 访问 [AnythingLLM 官网](https://anythingllm.com/)
2. 下载对应系统的安装包
3. 安装并启动

### 第二步：配置模型

1. 点击设置图标
2. 选择"LLM 首选项"
3. 配置你的 API Key（支持 OpenAI、Claude、DeepSeek 等）

**或者使用本地模型**：
1. 先安装 Ollama
2. 在 AnythingLLM 中选择 Ollama 作为 LLM 提供商
3. 选择模型（如 qwen2.5:7b）

### 第三步：创建知识库

1. 点击"上传文档"
2. 拖入你的 PDF、Word、TXT 文件
3. 等待系统自动向量化
4. 创建工作空间并关联知识库

## 📚 核心功能详解

### 1. 文档知识库（最实用）

**使用场景**：把文档变成 AI 可以理解的知识库。

**支持的文档格式**：
- PDF
- Word (.docx)
- TXT
- Markdown
- 网页 URL

**操作步骤**：
1. 点击"上传文档"
2. 选择文件或输入 URL
3. 等待系统自动向量化
4. 在对话中引用知识库

**实用技巧**：
- 把你的写作素材、参考资料都上传
- 为不同主题创建不同知识库
- 定期更新知识库，保持素材新鲜

### 2. 工作空间管理

**使用场景**：按项目/主题管理写作会话。

**功能**：
- 创建多个工作空间
- 每个工作空间独立的知识库和对话历史
- 不同工作空间可以使用不同模型

**操作步骤**：
1. 点击"创建工作空间"
2. 命名工作空间（如：小说创作、论文写作）
3. 关联知识库
4. 开始对话

**实用技巧**：
- 为每个写作项目创建独立工作空间
- 在工作空间中设置不同的系统提示词
- 定期清理不需要的工作空间

### 3. 写作辅助

**使用场景**：边写边问 AI，不打断创作流程。

**功能**：
- 选中文本，右键询问 AI
- 快速获取解释、改写、扩展
- 不用切换窗口

**实用技巧**：
- 写作时遇到不确定的地方，选中直接问 AI
- 让 AI 帮你检查语法和表达
- 让 AI 给出改写建议

### 4. 自定义 Agent

**使用场景**：配置 AI 代理自动执行任务。

**功能**：
- 自动搜索资料
- 自动总结文档
- 自动创作内容

**操作步骤**：
1. 点击"Agent"标签
2. 配置 Agent 的能力和工具
3. 给 Agent 分配任务
4. 查看 Agent 的执行结果

## 💡 实战案例：基于知识库写文章

### 步骤一：准备素材

1. 收集你的参考资料（PDF、网页、笔记）
2. 上传到 AnythingLLM 知识库

### 步骤二：创建工作空间

1. 创建"文章写作"工作空间
2. 关联你的知识库
3. 设置系统提示词（如：你是一个专业的写作助手）

### 步骤三：开始写作

1. 在工作空间中开始对话
2. 告诉 AI 你想写什么主题
3. AI 会基于知识库中的素材给出建议

### 步骤四：完善输出

1. 让 AI 帮你扩展内容
2. 让 AI 帮你润色语言
3. 导出为 Markdown 或其他格式

## ⚠️ 使用注意事项

### 避坑指南

1. **初次加载时间**
   - 首次启动需要加载模型，可能需要几分钟
   - 耐心等待，不要重复启动

2. **内存占用**
   - 使用本地模型会占用较多内存
   - 建议 16GB 以上内存

3. **知识库质量**
   - 向量化质量取决于文档质量
   - 建议使用格式规范的文档

4. **成本控制**
   - 使用云端 API 会产生费用
   - 建议先用本地模型测试

## 🔧 常见问题解答

**Q：需要编程基础吗？**
A：不需要。AnythingLLM 是图形界面，点击就能使用。

**Q：支持哪些大模型？**
A：支持 OpenAI GPT-4、Claude、DeepSeek、Ollama 本地模型等。

**Q：数据安全吗？**
A：数据存储在本地，非常安全。使用本地模型时，数据完全不出电脑。

**Q：免费吗？**
A：软件本身免费开源，但使用云端 API 会产生费用。
