## 📚 核心功能详解

### 1. 知识库工作流（最实用）

**使用场景**：可视化拖拽构建知识库处理流程。

**操作步骤**：
1. 点击"创建应用"→"工作流"
2. 拖拽节点构建流程
3. 连接节点，设置参数
4. 测试并发布

**典型流程**：
```
数据导入 → 文本预处理 → 向量化 → 问答匹配 → 内容生成
```

**实用技巧**：
- 先用简单流程测试，确认每步都正确
- 逐步增加复杂度
- 使用"条件分支"处理不同情况

### 2. RAG 增强写作

**使用场景**：让 AI 基于你的专属知识库进行创作。

**操作步骤**：
1. 上传你的参考素材（PDF、Word、网页等）
2. 等待系统自动向量化
3. 创建应用并关联知识库
4. 开始对话，AI 会基于你的素材回答

**实用技巧**：
- 把你的写作素材、参考资料都上传
- 定期更新知识库，保持素材新鲜
- 为不同主题创建不同知识库

### 3. 多格式导入

**支持的格式**：
- PDF
- Word (.docx)
- TXT
- Markdown
- CSV
- 网页 URL
- 手动输入

**操作步骤**：
1. 点击"知识库"→"导入数据"
2. 选择文件或输入 URL
3. 等待系统处理完成

**实用技巧**：
- 使用格式规范的文档，向量化质量更高
- 大文件建议分段上传
- 定期清理不需要的数据

### 4. 内容生成模板

**内置模板**：
- 写作模板：文章大纲、段落扩展、风格转换
- 翻译模板：中英互译、学术翻译
- 总结模板：文档摘要、会议纪要
- 润色模板：语法修正、表达优化

**自定义模板**：
1. 点击"模板管理"
2. 创建你的模板
3. 设置提示词和参数
4. 保存并使用

## 💡 实战案例：构建团队写作知识库

### 步骤一：准备素材

1. 收集团队的历史文章、参考资料
2. 整理成规范的文档格式

### 步骤二：创建知识库

1. 在 FastGPT 中创建知识库
2. 上传所有素材
3. 等待系统向量化完成

### 步骤三：创建工作流

1. 创建"文章生成"工作流
2. 设置输入（文章主题）
3. 连接知识库检索节点
4. 连接内容生成节点
5. 设置输出格式

### 步骤四：团队使用

1. 分享应用给团队成员
2. 团队成员输入主题，自动生成文章
3. 检查并修改输出内容
4. 发布到各平台

## ⚠️ 使用注意事项

### 避坑指南

1. **硬件要求**
   - 本地部署需要较好的硬件
   - 建议 16GB 以上内存
   - 有 GPU 更好

2. **知识库质量**
   - 向量化质量取决于文档质量
   - 建议使用格式规范的文档

3. **成本控制**
   - 使用云端 API 会产生费用
   - 建议先用免费额度测试

4. **数据安全**
   - 云服务数据会经过第三方服务器
   - 敏感数据建议本地部署

## 🔧 常见问题解答

**Q：需要编程基础吗？**
A：不需要。FastGPT 是可视化界面，拖拽就能构建应用。

**Q：和 Dify 有什么区别？**
A：FastGPT 更专注于知识库场景，Dify 功能更全面。如果你主要需要知识库功能，FastGPT 更合适。

**Q：支持哪些大模型？**
A：支持 OpenAI GPT-4、Claude、DeepSeek、Qwen 等主流模型。

**Q：免费吗？**
A：云服务有免费额度，超出需要付费。本地部署完全免费，但需要自己准备服务器和 API Key。
