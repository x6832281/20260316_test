## Project Overview

**AI 萌新小窝** — a Chinese static website educating AI beginners. Vite-powered SSG (Static Site Generator) with CSS/JS code splitting and article pre-rendering. Deployed at https://corely.top.

## Architecture

### Build Pipeline (Vite SSG)
The site uses a 4-step build pipeline (`npm run build`):
1. **`node scripts/validate.mjs`** — validates `data/manifest.json` for ID conflicts and missing files
2. **`vite build`** — Vite multi-page build from `src/` → `dist/` with CSS/JS code splitting and hashing
3. **`node scripts/generate-articles.mjs`** — reads Vite-processed `dist/article.html` as template, renders all 82 markdown articles using `marked`, outputs to `dist/article/{id}.html`
4. **`py generate_feeds.py`** — regenerates `rss.xml` and `sitemap.xml`

Build output goes to `dist/` (pure static files, Nginx-deployable).

### Content discovery: manifest.json (primary)

`data/manifest.json` is the single source of truth for discovering `.md` content files. It maps file paths to metadata (title, summary, date, category, dir). Client-side JS reads it via `fetchManifest()`.

**When to regenerate**: Run `py build_manifest.py` whenever `.md` files are added, removed, or renamed.
- On Windows, use `py` (Python launcher) instead of `python` to avoid the Windows Store stub issue.

**Feed generation**: Run `py generate_feeds.py` to regenerate `rss.xml` and `sitemap.xml`.

### Article Pre-rendering (SSG)
- Template: `src/article.html` (with `{{PLACEHOLDERS}}` for content injection)
- Script: `scripts/generate-articles.mjs` reads Vite-processed `dist/article.html` (has asset hashes), injects markdown-rendered HTML, writes to `dist/article/{id}.html`
- Backward compatibility: Nginx rewrites `article.html?id=xxx` → `article/xxx.html` (see `nginx-ssg.conf`)
- Dual-registration pattern still applies: `data/manifest.json` + `ARTICLES_MAP` in `src/scripts/article.js`

### PWA
- `service-worker.js` — cache version `kaka-cache-v20260517-ssg`, network-first for HTML, cache-first for assets
- `manifest.json` (root) — PWA manifest

### External services
- **Supabase** — used by board.html and disclaimer.html message board. Table: `message` (columns: name, message, type, created_at). SDK from CDN.
- **Baidu Analytics** — `hm.js?7da04a23e1fb1934e90419a677b100e2` on every page.
- **Google Fonts** — Noto Serif SC + Noto Sans SC, async loaded.

## URL patterns and conventions
- Article detail links: `article.html?id={content-id}` (Nginx rewrite: `article/{id}.html`)
- Pre-rendered articles: `article/{id}.html` (e.g. `/article/book-analysis-001.html`)
- All asset paths are absolute (`/scripts/main.js`, `/assets/main-xxx.css`)

## Workflow

### Local development
```bash
npm run dev       # Vite dev server with HMR
npm run build     # Full build pipeline
npx vite preview  # Preview production build
```

### Adding content
1. Add `.md` file to `data/{category}/`
2. Run `py build_manifest.py` to update `data/manifest.json`
3. Add mapping in `src/scripts/article.js` → `ARTICLES_MAP`
4. Run `npm run build`

### Content ID naming
- `{category}-{number}`: newbie-001, deai-001, book-analysis-001, littheory-001, book-excerpt-001
- `{category}-{slug}`: knowledge-001-gpt-academic, project-001-awesome-chatgpt-prompts

### Content tags and accent classes
- `tag-ai` / `accent-ai` — AI topics (blue accent)
- `tag-game` / `accent-game` — game dev
- `tag-resource` / `accent-resource` — resources

## Notable implementation details

- `src/scripts/article.js` exports `ARTICLES_MAP` via `module.exports` for Node.js validate.mjs to import
- `DIR_ID_PREFIX` is duplicated in `src/scripts/main.js`, `src/scripts/article.js`, `scripts/get-article-id.mjs`, and `generate_feeds.py` — keep in sync
- Markdown rendering uses `marked` library (Node.js build-time) and a fallback regex parser in `article.js` (client-side for dev mode)
- CSS uses CSS custom properties in `:root` — changing variables cascades to all components
- Mobile breakpoints: 1024px (tablet), 768px (mobile), 480px (small mobile)
- iOS zoom prevention: form inputs use `font-size: 16px`
- `prefers-reduced-motion` supported for accessibility
